```
┌─────────────────────────────────────────────────────────────┐
│              優化前：標準 MLP 架構 (1.44M 參數)              │
├─────────────────────────────────────────────────────────────┤

│  Input [B*S, 360]                                          │
```

### 層級 3: 批次大小自適應

```
┌─────────────────────────────────────────────────────────────┐
│              優化前：靜態估計 (保守，batch=64)               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  估計公式：                                                 │
│  batch_size = (VRAM_budget - margin) / sample_bytes        │
│  = (12GB - 1.5GB) / sample_bytes                           │
│  = 64  ← 估計不精確                                         │
│                                                             │
│  問題：                                                     │
│  • GPU VRAM 利用率 <50%                                    │
│  • 未考慮激活值、臨時緩衝、邊界情況                         │
│  • AMP (bf16) 混合精度節省未充分利用                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘

             ↓↓↓ 改進方案 ↓↓↓

┌─────────────────────────────────────────────────────────────┐
│            優化後：運行時二分查找 (batch=128-256)            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  二分查找邏輯：                                             │
│                                                             │
│  低=32   高=512                                             │
│    ↓                                                        │
│  測試 mid=256                                               │
│    ├─ 估計記憶體 < 預算 ✓                                   │
│    ├─ 實際測試 forward ✓                                    │
│    └─ 成功！低=256                                          │
│      ↓                                                      │
│  測試 mid=384                                               │
│    ├─ 估計記憶體 > 預算 ✗                                   │
│    └─ 失敗 OOM，高=384                                      │
│      ↓                                                      │
│  ... (8-10 次迭代)                                          │
│      ↓                                                      │
│  最終：batch_size = 192  ← 實際最大值                       │
│                                                             │
│  結果：                                                     │
│  • GPU 利用率 ↑ 90%                                         │
│  • 吞吐量提升 3x (64 → 192)                                 │
│  • 自動適應不同模型/數據大小                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 風險和回退策略

```
┌─────────────────────────────────────────────────────────────┐
│                      風險管理                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  風險 1: 精度下降 (新模型)                                  │
│  ├─ 檢測：Sharpe 值下降 >5%                                │
│  ├─ 回退：保留原版模型，A/B 對比                           │
│  └─ 緩解：確保特徵嵌入維度足夠 (≥16)                       │
│                                                             │
│  風險 2: OOM (自適應批次)                                  │
│  ├─ 檢測：訓練中 RuntimeError                              │
│  ├─ 回退：catch 異常，自動降低 batch size 50%             │
│  └─ 緩解：包含 OOM 回退邏輯在 trainer                      │
│                                                             │
│  風險 3: 數據不一致 (Panel 增量)                           │
│  ├─ 檢測：行數/列數發生變化                                 │
│  ├─ 回退：force_rebuild=True 重建緩存                      │
│  └─ 緩解：完整的時間戳和哈希檢查                           │
│                                                             │
│  風險 4: 進程崩潰 (多 fold 並行)                           │
│  ├─ 檢測：fold 結果缺失                                    │
│  ├─ 回退：重新訓練該 fold，或使用檢查點                   │
│  └─ 緩解：每個 fold 獨立保存結果                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
