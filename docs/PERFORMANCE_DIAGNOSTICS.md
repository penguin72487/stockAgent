```
│      date_idx = valid_indices[index]  # [0, T_fold)                      │

│          "x": features[start_idx:date_idx+1],  # [lookback, S, F]       │
│          "future_log_returns": returns[date_idx],  # [S]                 │
│          "tradable_mask": mask[date_idx],  # [S]                         │
│          "benchmark": benchmark[date_idx],  # scalar                     │

│                                                                           │
│  批量示例：                                                                │
│  Batch Size = 32, lookback = 1, S = 100, F = 12                         │
│  ├─ x:                   [32, 1, 100, 12]  = 48 KB                      │
│  ├─ future_log_returns:  [32, 100]         = 12.8 KB                    │
│  ├─ tradable_mask:       [32, 100]         = 12.8 KB (bool)             │
│  └─ benchmark:           [32]              = 256 B                       │
│                                                                           │
│  ✅ 無 look-ahead bias (驗證無誤)                                         │
│  ⚠️ 缺少特徵標準化驗證（見下文）                                         │
│                                                                           │
└──────────────────────────────────────────────────────────────────────────┘
                    ↓ [CrossSectionalMLP]
┌──────────────────────────────────────────────────────────────────────────┐
│                     🧠 模型層 (Forward Pass)                              │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                           │
│  輸入：[B=32, lookback=1, S=100, F=12]                                   │
│                                                                           │
│  步驟 1：特徵嵌入                                                          │
│  ├─ 線性層：F=12 → embedding_dim=64                                      │
│  ├─ 輸出：[B*S, lookback, 64] = [3200, 1, 64]                           │
│  └─ 計算：32 × 100 × 12 × 64 ≈ 2.4M FLOPs                               │
│                                                                           │
│  步驟 2：Transformer 編碼器 (Flash-Attention v2)                        │
│  ├─ 配置：2 層, 8 注意力頭, hidden=256                                   │
│  ├─ 輸入：[3200, 1, 64]  (lookback=1 時無用!)                            │
│  ├─ ⚠️ 問題：lookback=1 時，自注意力無法提取時序特徵                     │
│  │       應該有條件邏輯：                                                 │
│  │       if lookback > 1:                                                │
│  │           use transformer                                            │
│  │       else:                                                           │
│  │           use mlp                                                     │
│  └─ 輸出：[3200, 1, 64]                                                  │
│                                                                           │
│  步驟 3：池化                                                              │
│  ├─ 取最後時間步：x[:, -1, :] = [3200, 64]                              │
│  └─ lookback=1 時 = 取唯一時間步 (無實際池化)                           │
│                                                                           │
│  步驟 4：投資組合評分頭                                                    │
│  ├─ 線性層：64 → 1                                                       │
│  ├─ 輸出：[3200, 1]，reshape 到 [32, 100]                                │
│  └─ 掩膜 Softmax：強制不可交易權重 = 0                                   │
│                                                                           │
│  最終輸出：[B, S] = [32, 100]  (投資組合權重)                             │
│                                                                           │
│  性能：GPU 內存 ≈ 50-100 MB (激活值)                                    │
│        吞吐量 ≈ 1000+ 樣本/秒 (Flash-Attn)                               │
│                                                                           │
└──────────────────────────────────────────────────────────────────────────┘
                      ↓ [Sharpe Loss]
┌──────────────────────────────────────────────────────────────────────────┐
│                     📉 損失層                                             │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                           │
│  計算步驟：                                                                │
│  1. 權重標準化：[B, S] → [B, S]                                          │
│     normalized = weight / weight.sum(dim=1)                              │
│                                                                           │
│  2. 組合回報：[B]                                                         │
│     portfolio_return = (normalized * future_returns).sum(dim=1)          │
│                                                                           │
│  3. Sharpe 計算：scalar                                                  │
│     mean_ret = portfolio_return.mean()                                   │
│     std_ret = portfolio_return.std()                                     │
│     sharpe = mean_ret / std_ret * sqrt(252)  [✅ 已修復梯度穩定性]       │
│                                                                           │
│  4. 換手成本：[B]                                                         │
│     turnover = |weight_t - weight_{t-1}|.sum(dim=1)                     │
│     turnover_cost = mean(turnover) * fee_per_side                        │
│                                                                           │
│  5. 複合損失：scalar                                                      │
│     loss = -gamma_sharpe × sharpe + gamma_turnover × turnover_cost      │
│                                                                           │
│  ✅ 此層已優化，梯度穩定                                                  │
│                                                                           │
└──────────────────────────────────────────────────────────────────────────┘
                    ↓ [Backward Pass]
┌──────────────────────────────────────────────────────────────────────────┐
│                     🔄 反向傳播                                           │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                           │
│  loss.backward()  ← 計算 ∂loss/∂param                                    │
│                                                                           │
│  ⚠️ 問題：自動微分圖持留在內存中                                         │
│  症狀：                                                                    │
│  - 第一個 epoch：VRAM 使用 ≈ 300 MB                                     │
│  - 第二個 epoch：VRAM 使用 ≈ 400 MB                                     │
│  - 第十個 epoch：VRAM 使用 ≈ 1.2 GB                                     │
│  - 結果：OOM 或速度下降                                                   │
│                                                                           │
│  修復：添加 optimizer.zero_grad()  ← 顯式清理                            │
│                                                                           │
└──────────────────────────────────────────────────────────────────────────┘
```

## 6️⃣ 特徵標準化的重要性

### Transformer 注意力穩定性

```
案例：未標準化特徵

輸入數據分佈：
Feature 1: [0.0001 - 0.001]      (範圍很窄)
Feature 2: [-1000 - 1000]         (範圍很寬)
Feature 3: [0.5 - 2.5]            (中等範圍)

嵌入層處理：
E = F @ W  (W 初始化為 N(0, 1))

問題：
├─ Feature 2 的嵌入 = 1000 × (中等) = 大數值
├─ Feature 1 的嵌入 = 0.0001 × (中等) = 小數值
└─ 數值範圍不平衡 → Softmax 發散 → 梯度爆炸

Softmax 計算：
a_ij = exp(q_i @ k_j / sqrt(d))

若 q_i @ k_j 包含 [1e-5 ~ 1e3] 的混合：
├─ exp(1e3) ≈ 無窮 → 上溢 (overflow)
├─ exp(1e-5) ≈ 1 → 權重崩潰到 0
└─ 結果：NaN 或數值不穩定

標準化后：
Feature all: [-2 - 2]  (統一範圍)
├─ 嵌入：[-64 - 64]
├─ Softmax 輸入：[-64 - 64] / sqrt(64) = [-8 - 8]
├─ exp 輸入：[-8 - 8] → 安全範圍
└─ 結果：穩定、有意義的權重

驗證方法：
print(features.mean(), features.std())  # 應近似 (0, 1)
```

## 7️⃣ 訓練循環優化前後對比

### 自動微分圖管理

```
❌ 未修復：

for epoch in range(1000):
    for batch in train_loader:
        y = model(x)
        loss = loss_fn(y, target)
        loss.backward()  ← 建立計算圖
        optimizer.step()
        # ⚠️ 缺少清理！計算圖仍在 GPU 內存中

VRAM 增長曲線：
│
│     ╱╱╱
│   ╱╱
│  ╱  
│ ├─────→ Epoch

結果：OOM 或速度降低

✅ 修復后：

for epoch in range(1000):
    for batch in train_loader:
        with autocast(dtype=amp_dtype):
            y = model(x)
            loss = loss_fn(y, target)
        
        scaler.scale(loss).backward()
        scaler.unscale_(optimizer)
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        scaler.step(optimizer)
        scaler.update()
        
        optimizer.zero_grad()  ← ✅ 顯式清理
        
        # 計算圖在內存中釋放

VRAM 使用曲線：
│
│ ─────────────  ← 穩定在基線
│  
│ ├─────→ Epoch

結果：內存穩定，速度一致
```
